你这个现象我完全认同，而且从你给的三张图上**肉眼就能看出来**：

* **第三张传统算法**：背景杂质压得更干净，玻璃后的树叶纹理被明显“降复杂度”，但“茂石”字样和主体边缘还比较稳。
* **第一张模型结果**：背景高频还残留不少，说明“该滤的地方没滤够”；同时边缘和文字周围又有被抹软的感觉，说明“该保的地方又保坏了”。

这正是**PSNR / SSIM 很容易失灵**的典型场景：
它们更偏“信号保真”，但你这个任务本质上是**有选择地改变图像**：

* 背景希望**主动变得更平滑**
* 边缘希望**尽量不变**
* 而且这种“该变哪里、不该变哪里”还和结构区域有关

所以即使 PSNR、SSIM 很高，主观也可能不对。这个结论和大量 IQA 研究是一致的：PSNR、SSIM 往往更关注信号误差或简单结构统计，对感知质量、人眼偏好和复杂失真类型的相关性并不总是好。([cns.nyu.edu][1])

# 一、最值得你先试的通用指标

---

## 1）DISTS：我最推荐你先上这个

**优先级：最高**

DISTS 的核心思想就是把**结构 similarity** 和 **纹理 similarity** 分开建模再统一起来。它专门强调：传统 FR-IQA 往往对纹理替换、纹理重采样太敏感，而人眼并不总是这样看；DISTS 则显式地对结构和纹理做统一建模。这个特点跟你现在的任务特别像，因为你本来就是要“背景纹理降掉，但结构不能坏”。([cns.nyu.edu][2])

### 为什么它适合你

你这个问题不是“整张图越像 reference 越好”，而是：

* 背景纹理可以不同一点，甚至应该更简单
* 但结构边缘不能乱

这正是 DISTS 比 PSNR / SSIM 更有机会抓住的东西。([cns.nyu.edu][2])

### 你怎么用

把**传统滤波算法输出**当 reference，模型输出当 test：

* `DISTS(model, traditional)`

通常：

* **越小越好**（表示越接近传统参考的感知结构/纹理）

### 我对你这个任务的判断

如果第一张真的出现了“边缘也被糊了、背景又没压够”的情况，
**DISTS 大概率比 PSNR/SSIM 更容易把这种差异拉出来。**

---

## 2）LPIPS：很适合补充“主观像不像”

LPIPS 是基于深层特征的感知距离指标，论文基于大规模人类感知判断数据验证，结果显示深特征距离比许多传统指标更符合人类感知相似性。([CVF开放获取][3])

### 为什么适合你

你现在最痛苦的就是：

> “数值看着不错，但肉眼一看不对。”

LPIPS 解决的就是这个方向的问题，它更偏“看起来像不像人会接受的视觉结果”。

### 你怎么用

还是把传统算法输出作为 reference：

* `LPIPS(model, traditional)`

通常：

* **越小越好**

### 注意

LPIPS 很有用，但它有一个特点：

* 它是“深特征感知距离”
* 不一定专门对“边缘保留 vs 背景抑制”这个任务对齐得最完美

所以**它适合做补充，不建议单独做唯一主指标**。([arXiv][4])

---

## 3）HaarPSI：非常值得试，尤其适合你这种“边缘+感知”场景

HaarPSI 是基于 Haar 小波的全参考感知相似性指标，论文强调它**计算便宜**，而且在多个数据库上对主观评分的相关性很强。([arXiv][5])

### 为什么适合你

Haar 小波本身就对：

* 边缘
* 局部结构
* 多尺度变化

比较敏感。

你这种图像里，真正重要的主观质量差异往往正出在：

* 文字轮廓
* 构筑物边缘
* 主体轮廓带
* 背景高频压制后的局部感知变化

HaarPSI 往往会比 SSIM 更贴近这些变化。([arXiv][5])

### 你怎么用

* `HaarPSI(model, traditional)`

通常：

* **越大越好**

---

## 4）FSIM：经典而且很贴你这个任务

FSIM 的核心是：

* **Phase Congruency（相位一致性）**：抓显著结构
* **Gradient Magnitude（梯度幅值）**：抓局部对比和边缘变化

论文的出发点就是：人视觉主要依赖低层特征，而不是单纯像素误差。([www4.comp.polyu.edu.hk][6])

### 为什么适合你

你这个场景就是典型的：

* 结构边缘很重要
* 背景纹理不一定要精确保真

FSIM 本身对结构显著性和梯度都敏感，所以也很合适。

### 你怎么用

* `FSIM(model, traditional)`

通常：

* **越大越好**

---

## 5）GMSD：专门看“梯度相似性”的，抓边缘发糊很好用

GMSD 的核心是看**梯度幅值相似性**，再看这种局部质量图的全局波动。论文强调它高效，而且对感知质量有竞争力。([arXiv][7])

### 为什么适合你

你现在一个核心问题就是：

> 不该滤的边也一起滤波了

边被糊，本质上就是**梯度被削弱**。
GMSD 对这种现象通常比较敏感。

### 你怎么用

* `GMSD(model, traditional)`

通常：

* **越小越好**

### 适合扮演的角色

它非常适合做：

* **边缘退化报警指标**
* 或者你总分里的一个分项

---

## 6）VIF：可以作为“信息保真”补充项

VIF 是 Visual Information Fidelity，核心是用信息论方式衡量失真图相对参考图保留了多少可用视觉信息。([德克萨斯大学奥斯汀分校图像与视频工程实验室][8])

### 为什么你可以试

如果你怀疑模型有一种情况是：

* 背景没压对
* 边缘也掉信息
* 整体“信息组织”变差

那 VIF 有时比单纯 SSIM 更能体现这个问题。

### 但优先级

我会把它放在：

* DISTS / LPIPS / HaarPSI / FSIM / GMSD 之后

# 四、我最推荐你自己加的 4 个任务定制指标

---

## 指标 1：背景高频残差误差

### 目标

专门衡量：

> “模型在低结构背景区域，到底有没有把不重要高频压下去？”

### 做法

先用传统算法里的结构图，得到一个**低结构 mask**：

* `M_bg = 1` 表示背景低结构区
* `M_bg = 0` 表示非背景区

然后只在这个区域里比较**高频能量**。

例如：

```python
HF(x) = abs(Laplacian(x)) 
```

或者 Sobel 高频能量：

```python
HF(x) = sqrt(Sobel_x(x)^2 + Sobel_y(x)^2)
```

然后定义：

```python
E_bg = mean( M_bg * | HF(model) - HF(traditional) | )
```

### 解释

如果模型背景没滤够，那背景里还残留很多高频杂质：

* `HF(model)` 会偏大
* 和 `HF(traditional)` 的差异也会变大
* 所以 `E_bg` 会变大

### 这个指标很贴你现在的问题

因为你现在主观上最不满意的一点，就是：

> “背景不重要的滤波不充分”

这个指标就是专门打它的。

---

## 指标 2：边缘保真误差

### 目标

专门衡量：

> “强结构边缘有没有被模型一起抹掉？”

### 做法

还是利用结构图，拿到**高结构区域 mask**：

* `M_edge`

或者直接从传统输出 / 原图提边缘带：

* Canny
* Sobel
* 结构图阈值化后膨胀出边缘带

然后只在边缘带内比较梯度：

```python
G(x) = sqrt(Sobel_x(x)^2 + Sobel_y(x)^2)
```

定义：

```python
E_edge = mean( M_edge * | G(model) - G(traditional) | )
```

或者直接用局部 gradient similarity。

### 解释

如果你的模型把边缘抹软了：

* `G(model)` 会比 `G(traditional)` 小
* `E_edge` 就会上升

### 这很适合你现在的痛点

你现在最想抓住的第二个问题就是：

> “不该过滤的边也滤波了”

这个指标就是单独盯边的。

---

## 指标 3：区域加权感知指标

### 目标

把 DISTS / LPIPS / HaarPSI 从“整图平均”升级成“分区加权”。

### 做法

不要只算整图：

```python
DISTS_all
```

改成：

```python
DISTS_fg = DISTS(model * M_edge, traditional * M_edge)
DISTS_bg = DISTS(model * M_bg, traditional * M_bg)
```

或者做 patch 级统计后分别平均。

最后加权：

```python
Score = w1 * DISTS_fg + w2 * DISTS_bg
```

但这里要注意方向：

* DISTS / LPIPS 是越小越好
* HaarPSI / FSIM 是越大越好

### 我建议的加权逻辑

如果你更怕“边被抹坏”，那边缘权重就高一点，比如：

* `w_fg = 0.7`
* `w_bg = 0.3`

如果你们当前更关心“码率收益不达标，因为背景没压够”，那就反过来。

---

## 指标 4：边缘带 LPIPS / GMSD

### 目标

只在边缘附近看主观差异。

### 做法

先做一个边缘带 mask，例如：

* 对高结构区域膨胀 3~5 像素，形成 `M_band`

然后：

```python
LPIPS_edge = LPIPS(model ⊙ M_band, traditional ⊙ M_band)
GMSD_edge = GMSD(model ⊙ M_band, traditional ⊙ M_band)
```

### 价值

整图 LPIPS 往往会被大面积背景平均掉。
但你真正关心的是**边缘附近看起来对不对**。

所以“边缘带 LPIPS / GMSD”通常比整图更能抓你现在这种问题。

# 五、我建议你最终采用的“总评分系统”

你真正要的不是一个论文里现成万能值，而是一个**能把你这类主观失败案例数值化分开**的体系。

我建议你做成下面这样：

## 方案 1：简单实用版

### 主指标

* **DISTS**
* **GMSD**
* **背景高频残差误差 `E_bg`**
* **边缘保真误差 `E_edge`**

### 解释

* DISTS：总感知
* GMSD：边缘梯度
* `E_bg`：背景抑制够不够
* `E_edge`：边缘有没有被抹坏

---

## 方案 2：你这个任务最推荐版

定义一个总分：

```text
TotalScore
= a * (1 - norm(DISTS))
+ b * norm(HaarPSI)
+ c * (1 - norm(LPIPS_edge))
+ d * (1 - norm(E_bg))
+ e * (1 - norm(E_edge))
```

其中：

* `LPIPS_edge`：边缘带 LPIPS
* `E_bg`：背景高频误差
* `E_edge`：边缘梯度误差

权重建议一开始先试：

* `a=0.20`
* `b=0.15`
* `c=0.25`
* `d=0.20`
* `e=0.20`

然后拿一批你们肉眼排序过的 case 去调。
