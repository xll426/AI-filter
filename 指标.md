# Prefilter 指标与 Loss 方案

## 1. 任务目标

这个任务不是普通的“整图尽量接近 `ref`”。

真实目标是两件事同时成立：

1. 背景低价值区域要尽量接近 `ref.py` 的滤波效果。
2. 主体结构和重要边缘要尽量保住，不能为了追 `ref` 把边一起抹掉。

所以：

- `PSNR/SSIM` 只能作为辅助参考。
- 最终验证主指标要按“背景对 `ref`、边缘对 `ori`”来定义。

---

## 2. 为什么不能把 `selective_score` 直接当训练主 loss

不建议。

原因不是这个指标不重要，而是它本身包含以下非平滑或离散步骤：

- 结构分数 `S` 的 percentile gating
- `bg_mask / edge_mask` 二值阈值
- `max/min/clip`
- CPU/Numpy 路径

这些步骤非常适合做验证评价，但不适合直接做反向传播主损失。

如果强行把它当主 loss，典型风险是：

1. 梯度不稳定，训练发散或抖动。
2. 模型学会投机地优化某几个统计量，而不是整体画质。
3. 早期训练阶段过分追“结构判别”，导致全局收敛变慢。
4. 不同 batch 的掩码分布波动过大，loss 曲线会很难看。

结论：

- `selective_score` 适合做验证主指标和 `best checkpoint` 选择标准。
- 训练反传仍然应该以稳定、可微、逐像素/感知型 loss 为主。

---

## 3. 指标逻辑

### 3.1 掩码来源

掩码不使用简单梯度阈值，而是复用 `ref.py` 的结构判断逻辑。

当前实现位置：

- [iqa_metrics_exact_refalgo_y.py](/mnt/d/fudan/prefilter_clean/iqa_metrics_exact_refalgo_y.py:465)
- [iqa_metrics_exact_refalgo_y.py](/mnt/d/fudan/prefilter_clean/iqa_metrics_exact_refalgo_y.py:521)
- [iqa_metrics_exact_refalgo_y.py](/mnt/d/fudan/prefilter_clean/iqa_metrics_exact_refalgo_y.py:540)

使用流程：

1. 从 `ori` 的 Y 分量出发。
2. 复现 `ref.py` 里的预处理和结构分数 `S`。
3. 用 `S < s_thr` 作为背景区。
4. 用 `S >= s_thr` 作为结构/边缘区。

默认：

- `mask_mode = detail_gain`
- `s_thr = 0.25`

这比简化版梯度 mask 更贴任务。

### 3.2 严格拟合型指标

这些指标是“和 `ref` 比”的严格型指标：

- `bg_hf_error`
  - 背景区高频误差，越小越好。
- `edge_preserve_error`
  - 结构区梯度误差，越小越好。
- `edge_over_smooth_ratio`
  - 结构区过平滑比例，越小越好。
- `edge_gmsd`
  - 结构区梯度相似性波动，越小越好。
- `bg_grad_energy_ratio`
  - 背景区梯度能量比，越接近 1 越好。
- `edge_grad_energy_ratio`
  - 结构区梯度能量比，越接近 1 越好。
- `structure_alignment_error`
  - 结构图对齐误差，越小越好。

这些指标回答的是：

“模型输出和 `ref` 到底有多像？”

### 3.3 任务型指标

真正用于选最优 checkpoint 的是任务型指标：

- 背景区对 `ref`
- 边缘区对 `ori`

定义如下。

#### 背景完成度

```text
bg_completion = 1 - bg_hf_error(pred, ref) / bg_hf_error(ori, ref)
```

解释：

- `1` 表示背景已经接近 `ref`
- `0` 表示和原图一样，基本没滤
- 小于 `0` 表示比原图还差

#### 边缘保持完成度

```text
edge_source_completion = 1 - edge_preserve_error(pred, ori) / edge_preserve_error(ref, ori)
```

解释：

- `> 0` 表示比 `ref` 更接近原图边缘
- `= 0` 表示和 `ref` 一样
- `< 0` 表示比 `ref` 还更伤边

#### 边缘保留比

```text
edge_retention_ratio = edge_grad_energy_ratio(pred, ori)
```

解释：

- 越接近 `1` 越好
- 明显小于 `1` 说明边缘能量掉了

#### 边缘过平滑比例

```text
edge_oversmooth_vs_src = edge_over_smooth_ratio(pred, ori)
```

解释：

- 越小越好

### 3.4 最终主指标

当前推荐主指标：

```text
selective_score =
100 * (
    0.45 * bg_completion
  + 0.35 * clip(edge_source_completion, -1, 1)
  + 0.10 * clip(edge_retention_ratio, 0, 1.2) / 1.2
  + 0.10 * max(0, 1 - edge_oversmooth_vs_src)
)
```

权重解释：

- `0.45` 给背景完成度
  - 因为前滤波本职工作就是“该滤的地方要滤掉”。
- `0.35` 给边缘保持完成度
  - 因为主体边不能坏。
- `0.10` 给边缘能量保留
  - 作为结构安全护栏。
- `0.10` 给过平滑惩罚
  - 防止模型把边磨平。

注意：

- 这个分数不是绝对满分型分数。
- `ref` 自己在这套定义下也不会是 `100`，因为它本身也会牺牲一部分边缘。

## 4. 训练与验证的职责分工

### 4.1 训练主 loss

当前推荐保持：

```text
L_total = 1.00 * L_charbonnier_roi(pred, ref)
        + 0.16 * L_ms_ssim(pred, ref)
```

对应代码：

- [loss.py](/mnt/d/fudan/prefilter_clean/loss.py:8)
- [loss.py](/mnt/d/fudan/prefilter_clean/loss.py:36)
- [train.py](/mnt/d/fudan/prefilter_clean/train.py:121)

其中：

- `L_charbonnier_roi`
  - 主收敛项
  - 保证模型稳定贴近监督目标 `ref`
  - ROI 区域权重大于非 ROI 区域
- `L_ms_ssim`
  - 感知/结构辅助项
  - 防止输出只会做像素平均

### 4.2 验证主指标

当前推荐：

- `primary_metric = selective_score`

原因：

- 训练 loss 负责稳定收敛
- 验证主指标负责“挑出真正更符合任务的模型”

这两者不要混成一个东西。

### 4.3 为什么这是更稳的方案

因为你的目标存在天然张力：

- 越贴 `ref`，背景往往越干净
- 但越贴 `ref`，边缘也可能被一起磨掉

所以：

- 训练阶段仍要靠稳定 loss 学出可用映射
- 验证阶段再用任务指标判断“哪个 checkpoint 更值得留”

这比直接用任务指标硬拉训练更稳。

## 5. 是否要把任务感知再加入训练 loss

可以，但只能小权重辅助，不能替掉主 loss。

推荐顺序：

### 阶段 A：先不加

先使用：

```text
训练 loss: Charbonnier + MS-SSIM
验证主指标: selective_score
best checkpoint: 按 selective_score 选
```

这是当前最推荐默认方案。

### 阶段 B：如果验证发现模型仍然“背景好了但边保不住”

再考虑加入一个很轻的辅助项，例如：

```text
L_total =
1.00 * L_charbonnier_roi(pred, ref)
+0.16 * L_ms_ssim(pred, ref)
+0.03 ~ 0.08 * L_edge_grad(pred, ori; edge_mask)
```

注意这里的辅助项只建议做：

- 小权重
- 后期微调
- 不替代主 loss

不建议一开始就上大权重任务感知 loss。

---

## 6. 当前最终建议

### 6.1 最终训练 loss

默认最终训练 loss：

```text
L_total = 1.00 * ROI-Charbonnier(pred, ref)
        + 0.16 * MS-SSIM(pred, ref)
```

### 6.2 最终验证指标

最终验证主指标：

```text
selective_score
```

同时保留以下辅助监控项：

- `bg_completion`
- `edge_source_completion`
- `edge_retention_ratio`
- `edge_oversmooth_vs_src`
- `psnr`
- `ssim`
- `val_loss`

### 6.3 最终选模标准

优先级建议：

1. 主标准：`selective_score` 最大
2. 安全护栏：`edge_source_completion >= 0`
3. 安全护栏：`edge_retention_ratio >= 0.90`
4. 同分时再参考 `PSNR/SSIM`
